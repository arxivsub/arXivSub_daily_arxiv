## 23. AnyGoal: Vision-Language Guided Multi-Agent Exploration for Training-Free Lifelong Navigation

**arXiv ID:** 2606.13878 | [PDF](https://arxiv.org/pdf/2606.13878v1)

**作者:** MoniJesu James `[一作]`, Dzmitry Tsetserukou `[通讯]`

**关键词:** `a1c26042-88d3-4e76-b403-2055e0dfc5c7` `c7913869-b026-40e7-b14b-dfd72dc55ea0` `edb9d762-f411-4838-a852-f2d638b018db` `4bf3b852-21ff-4736-b125-37e24f3c9a32` `90291a0e-9d36-4a08-9a16-89ce846d923f` `6b9ad54c-2d62-4a92-a500-d9cb644dd99c` `79276348-11e0-48e3-84bc-7ec231d0171c`

**🎯 论文内容**

提出了 AnyGoal，一种完全去中心化、无需训练的多机器人导航框架，利用 Vision–Language Model（VLM）与 2D 高斯贝叶斯价值图（BVM）实现前沿探索与多目标协调。

**💡 创新点**

创新点在于将 VLM 作为噪声后验融合进 BVM，实现前沿排序的 VLM-软最大与贝叶斯 UCB 的凸混合；同时通过共享 BVM 的精度加权融合实现多机器人间无中心调度的去中心化分配。

**🔧 技术方法**

技术包括开放词汇分割检测器（SAM3/GroundedSAM/YOLO‑World+MobileSAM）、空间 VLM（SpaceOM-3B）、深度投影的 GoalProjector、贝叶斯高斯更新、UCB 置信上界、Fast‑Marching‑Method（FMM）规划和基于前沿的分配策略。

**📊 数据集**

在 GOAT‑Bench 的 val_unseen 细粒度物理实验室（360 章节、2669 子任务）中进行评估，同时在真实 Unitree Go1 + 轮式平台中进行了硬件验证。

**📈 对比分析**

在严格物理实验室（离散 0.25 m 步长、42° HFOV、无瞬移）中，双机器人 AnyGoal 取得 52.4 % 的 Subtask Success Rate（比 Modular GOAT 提升 27.5 pp），单机器人 41.9 %；SPL 12.7 %；相较于 3D‑Mem、TANGO 等零样本基线均显著领先。

**⚠️ 局限性**

局限包括对透明、薄壁物体的感知失败、对精确地标对齐的依赖（需 Vicon/GPS），以及严格物理设置下 SPL 低于部分基线；单次评估导致实验不够统计充分。

## 357. tap: A File-Based Protocol for Heterogeneous LLM Agent Collaboration

**arXiv ID:** 2606.14445 | [PDF](https://arxiv.org/pdf/2606.14445v1)

**作者:** Minseo Kim `[一作]` `[通讯]` (HUA Labs), Minseo Kim (HUA Labs)

**关键词:** `1bc454a9-3d09-46c3-87e9-f7a9c36911df` `e2c980c8-7137-48ee-b99f-3fbde4cf81e7` `c59129cc-0f1d-4fee-85d8-abbb7eea50d6` `d0f189e1-0834-4ff4-b4e8-f515263ef669` `6b9ad54c-2d62-4a92-a500-d9cb644dd99c`

**🎯 论文内容**

提出了一种基于文件的协作协议，允许来自不同供应商的 LLM 代理（如 Claude 与 Codex）在不共享运行时或中心会话服务器的情况下，直接在同一代码库上进行开发与代码评审。

**💡 创新点**

创新点包括：① 采用文件优先设计，将 Markdown+YAML 形式的消息写入本地文件，兼容所有执行环境；② 结合实时通知（Claude 的 MCP 通道与 Codex 的 WebSocket）与文件检查路径，既能降低延迟，又能保证消息持久性与可回溯性；③ 通过 git worktree 实现工作隔离，避免文件冲突；④ 在真实生产仓库中开展长达 27 天、37 代的自应用实验，验证协议可行性与对多模型协作的实用价值。

**🔧 技术方法**

使用技术包括：文件系统消息存储、Markdown+YAML 元数据、git worktree、Node.js（npm 包 v0.5.2）、Claude MCP 通道、Codex WebSocket、Gemini 的文件轮询、自动化脚本处理配置与操作日志。

**📊 数据集**

实验数据集为一个包含 104 个文件、26,404 行代码与 76 个测试文件的生产仓库，在 27 天内产生 209 条 PR、717 条操作工件；另外收集了 375 条评审工件，用于计算缺陷检测率。

**📈 对比分析**

通过对 375 条评审工件进行统计，比较异构模型对（Claude-Codex）与同质模型对（Claude-Claude、Codex-Codex）的缺陷检测率，发现异构对检测率为 69.8%，同质对为 53.1%，表明模型多样性可提升评审效果；实验显示协议能够稳定运行且消息不会丢失。

**⚠️ 局限性**

主要限制：① 研究为观察性、非随机对照实验，评审分配存在偏差；② 评审工件可能重复计数，无法精确统计缺陷数；③ 仅在单一组织、单一仓库中验证，缺乏跨组织或跨语言推广验证；④ 实时协作仅涵盖 Claude 与 Codex，Gemini 仅实验性文件轮询；⑤ 37 代操作环境随时间演化，初期与后期条件不完全一致；⑥ LLM 性能随基础模型、工具使用能力和执行环境变化，可能影响结果。

**arXiv ID:** 2606.14459 | [PDF](https://arxiv.org/pdf/2606.14459v1)

**⚠️ 局限性**

局限性：样本量相对有限且合成语音比例高；持续学习效果对任务顺序敏感；仅评估了三种方法，未覆盖更广泛的 CL 技术；缺乏多语言和更真实场景下的验证，且对不同因子交互效应的深入分析不足。

## 467. Compressed Computation is (probably) not Computation in Superposition

**🎯 论文内容**

本文拆解并复现了压缩计算（Compressed Computation, CC）模型，证明其相较于仅实现 50 个 ReLU 函数的基线之所以能取得更低损失，关键在于模型内部噪声残差流产生的混合矩阵，而非真正实现“计算在叠加”，并基于该发现提出了新的半非负矩阵分解（SNMF）基线；

**💡 创新点**

主要创新在于揭示混合矩阵是 CC 模型性能提升的根本原因，并提出了仅依赖该混合矩阵的 SNMF 近似基线，能够在一定程度上复制原模型的损失曲线；

**🔧 技术方法**

**📊 数据集**

实验数据为 100 维稀疏输入，非零概率 p，取值区间 [-1,1]，并对不同噪声尺度（σ）下的混合矩阵进行评估；

**📈 对比分析**

与原始 CC 模型、仅实现 50 个 ReLU 函数的基线以及随机混合矩阵进行对比，结果显示在混合矩阵非零时损失显著下降，并随噪声尺度增大而进一步改善，SNMF 基线虽不能完全匹配训练模型，但在损失趋势上与之相近；

**⚠️ 局限性**

尽管揭示了混合矩阵的关键作用，但仍无法完整恢复训练模型的权重结构和损失表现，说明混合矩阵虽重要，但并未完全解释 CC 模型的全部优势，模型机制仍待进一步探究。
